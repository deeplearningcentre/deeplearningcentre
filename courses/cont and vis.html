12-Week Containerization & Virtualization Learning Plan by Deep Learning Centre Gurgaon

Week 1: Introduction to Virtualization and Containerization

Objective:

Understand the basics of virtualization and containerization, their differences, and use cases.

Topics Covered:

  • Introduction to Virtualization: Concepts, hypervisors (Type 1 and Type 2).
  • Introduction to Containerization: Concepts, advantages over traditional virtualization.
  • Comparison: Virtual Machines vs Containers.

Assignments:

  • Research and create a comparison chart for Virtual Machines vs Containers.
  • Install VirtualBox or VMware Workstation and explore basic virtual machine setup.

Week 2: Virtualization Technologies - Hypervisors and Virtual Machines

Objective:

Learn about hypervisors, virtual machines, and their setup in real environments.

Topics Covered:

  • Understanding Hypervisors: Type 1 (bare-metal) and Type 2 (hosted) hypervisors.
  • Creating and managing Virtual Machines (VMs) using VMware or VirtualBox.
  • Virtual Networking: Configuring virtual switches and network adapters for VMs.

Assignments:

  • Create a Virtual Machine using VirtualBox or VMware and install an OS of your choice.
  • Experiment with virtual networking: Set up a virtual network for VMs.

Week 3: Introduction to Docker

Objective:

Learn the basics of Docker, one of the most widely used containerization tools.

Topics Covered:

  • Introduction to Docker: Concepts, images, containers, and the Docker daemon.
  • Installing Docker: Setup Docker on your local machine (Windows, Mac, or Linux).
  • Basic Docker Commands: `docker run`, `docker ps`, `docker build`, and `docker images`.

Assignments:

  • Install Docker on your machine and run a "Hello World" container.
  • Explore Docker images and containers using `docker ps` and `docker images`.

Week 4: Docker Images and Dockerfile

Objective:

Learn how to create and manage custom Docker images using a Dockerfile.

Topics Covered:

  • Docker Images: Understanding how Docker images work.
  • Writing Dockerfile: How to create a custom Docker image from scratch.
  • Building and managing images with `docker build` and `docker push`.

Assignments:

  • Create a simple web app, write a Dockerfile for it, and build the image.
  • Push your custom image to Docker Hub.

Week 5: Docker Compose for Multi-Container Applications

Objective:

Learn how to manage multi-container applications using Docker Compose.

Topics Covered:

  • Introduction to Docker Compose: Define multi-container environments.
  • Docker Compose File (`docker-compose.yml`): Structure, services, volumes, and networks.
  • Managing Docker Compose: Commands such as `docker-compose up`, `docker-compose down`.

Assignments:

  • Create a simple multi-container application using Docker Compose (e.g., web app with a database).
  • Learn to scale containers using `docker-compose scale`.

Week 6: Networking in Docker Containers

Objective:

Understand networking concepts in Docker containers and how to configure networks between containers.

Topics Covered:

  • Docker Networking: Network drivers, bridge, host, overlay.
  • Linking Containers: Connect containers using networks in Docker.
  • Inspecting Networks: Use `docker network inspect` and other network-related commands.

Assignments:

  • Create a custom network in Docker and run multiple containers on it.
  • Experiment with linking containers and inspecting networks.

Week 7: Introduction to Kubernetes

Objective:

Understand the basics of Kubernetes, a powerful container orchestration tool.

Topics Covered:

  • Introduction to Kubernetes: Concepts, architecture (Pods, Nodes, Clusters).
  • Setting up a local Kubernetes cluster using Minikube or Docker Desktop.
  • Basic Kubernetes Commands: `kubectl` commands for managing pods, services, deployments.

Assignments:

  • Install Minikube or enable Kubernetes in Docker Desktop and start a cluster.
  • Run your first Kubernetes pod and explore `kubectl get pods`.

Week 8: Kubernetes Deployments and Services

Objective:

Learn how to deploy applications and expose them using Kubernetes deployments and services.

Topics Covered:

  • Deployments: How to create and manage deployments in Kubernetes.
  • Services: Exposing applications with Kubernetes Services (ClusterIP, NodePort, LoadBalancer).
  • Scaling applications: How to scale pods in Kubernetes.

Assignments:

  • Create a simple deployment in Kubernetes.
  • Expose the application using a Kubernetes Service and scale it up/down.

Week 9: Advanced Kubernetes Concepts

Objective:

Explore advanced features of Kubernetes, including persistent storage and ConfigMaps.

Topics Covered:

  • Persistent Volumes and Storage: Configuring persistent storage for applications in Kubernetes.
  • ConfigMaps and Secrets: Managing application configuration and sensitive data in Kubernetes.
  • Namespaces: Isolating resources within Kubernetes clusters using namespaces.

Assignments:

  • Set up persistent volumes and use them in a Kubernetes deployment.
  • Create and manage ConfigMaps and Secrets in Kubernetes.

Week 10: Monitoring and Logging in Containers

Objective:

Learn about container monitoring and logging to ensure applications are running smoothly.

Topics Covered:

  • Logging: Set up logging for containers using Docker logs and centralized logging solutions.
  • Monitoring: Use tools like Prometheus, Grafana for container monitoring.
  • Health Checks: Implement health checks for your Docker containers and Kubernetes pods.

Assignments:

  • Set up logging for your Docker containers using `docker logs` and explore centralized logging.
  • Monitor a Docker container using Prometheus and Grafana.

Week 11: Security in Containers

Objective:

Understand container security best practices and how to secure your Docker and Kubernetes environments.

Topics Covered:

  • Securing Docker Images: Scan images for vulnerabilities using tools like Clair or Trivy.
  • Securing Kubernetes: Authentication, RBAC, Network Policies.
  • Container Runtime Security: Configuring container runtimes with security best practices.

Assignments:

  • Scan your Docker images for vulnerabilities using a scanning tool like Trivy.
  • Implement Kubernetes RBAC policies and network policies for securing resources.

Week 12: Final Project and Recap

Objective:

Apply everything learned in a final project, covering containerization, Kubernetes, and security practices.

Topics Covered:

  • Final Project: Build and deploy a multi-container application using Docker, Kubernetes, and all concepts learned.
  • Recap: Review key concepts and address final questions or issues.

Assignments:

  • Complete your final project, demonstrating skills learned throughout the course.
  • Present your project and explain the architecture and security practices used.
