Docker Learning Plan by Deep Learning Centre Gurgaon

Week 1: Introduction to Docker and Containerization

Objective:

Understand the basic concepts of Docker and containerization.

Topics Covered:

  • What is Docker?
  • Containerization vs Virtualization
  • Docker Architecture: Images, Containers, Docker Daemon
  • Installing Docker on various platforms (Linux, Windows, macOS)

Assignments:

  • Install Docker on your machine.
  • Run your first "Hello World" container using `docker run`.

Week 2: Docker Images and Containers

Objective:

Learn how Docker images and containers work, and how to manage them.

Topics Covered:

  • Understanding Docker Images
  • Docker Containers: Running, stopping, and removing containers
  • Basic Docker Commands: `docker run`, `docker ps`, `docker stop`, `docker rm`
  • Building and running your own Docker images

Assignments:

  • Run multiple containers and manage them using basic Docker commands.
  • Create a custom Docker image and run a container from it.

Week 3: Dockerfile and Building Custom Images

Objective:

Learn how to create a Dockerfile to build custom Docker images.

Topics Covered:

  • What is a Dockerfile?
  • Creating Dockerfile instructions: `FROM`, `RUN`, `CMD`, `COPY`, `WORKDIR`
  • Building Docker images using `docker build`
  • Understanding best practices for writing Dockerfiles

Assignments:

  • Create a simple web application, write a Dockerfile for it, and build the image.
  • Optimize the Dockerfile by reducing the image size and improving build efficiency.

Week 4: Docker Compose

Objective:

Learn how to manage multi-container applications using Docker Compose.

Topics Covered:

  • Introduction to Docker Compose
  • Creating a `docker-compose.yml` file
  • Running multi-container applications with Docker Compose
  • Scaling applications using Docker Compose

Assignments:

  • Set up a multi-container application using Docker Compose (e.g., web app with a database).
  • Experiment with scaling containers and connecting them through Docker Compose.

Week 5: Docker Networking

Objective:

Learn how networking works in Docker and how to configure container communication.

Topics Covered:

  • Docker Networking Overview: Bridge, Host, and Overlay networks
  • Creating and managing custom networks in Docker
  • Linking containers with networking
  • Docker Network Inspection and Troubleshooting

Assignments:

  • Create a custom Docker network and connect containers on it.
  • Inspect and troubleshoot network configurations using Docker network commands.

Week 6: Docker Volumes and Data Persistence

Objective:

Understand how Docker handles data persistence through volumes.

Topics Covered:

  • Understanding Docker Volumes: Persistent data storage
  • Creating and using Docker volumes
  • Mounting host directories as Docker volumes
  • Docker data management and backups

Assignments:

  • Create a volume and mount it to a container.
  • Work with a database container and persist its data using Docker volumes.

Week 7: Docker Security Best Practices

Objective:

Learn security best practices for Docker images, containers, and the Docker daemon.

Topics Covered:

  • Docker Image Security: Scanning for vulnerabilities
  • Running Docker containers with limited privileges
  • Using Docker Content Trust (DCT)
  • Securing Docker Daemon and User Permissions

Assignments:

  • Scan Docker images for vulnerabilities using tools like Trivy or Clair.
  • Secure the Docker daemon by setting up user permissions and role-based access control (RBAC).

Week 8: Introduction to Docker Swarm

Objective:

Learn how to orchestrate Docker containers with Docker Swarm for cluster management.

Topics Covered:

  • What is Docker Swarm?
  • Initializing a Docker Swarm Cluster
  • Deploying and managing services in Docker Swarm
  • Scaling and updating services in a Swarm cluster

Assignments:

  • Initialize a Docker Swarm cluster with multiple nodes.
  • Deploy and manage services using `docker service` commands in Swarm mode.

Learn how to monitor and log Docker containers to ensure they run smoothly.
