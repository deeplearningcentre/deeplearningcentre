Kubernetes Learning Plan by Deep Learning Centre Gurgaon

Week 1: Introduction to Kubernetes

Objective:

Understand the basic concepts of Kubernetes and container orchestration.

Topics Covered:

  • What is Kubernetes?
  • Why use Kubernetes?
  • Components of Kubernetes: Pods, Nodes, Cluster
  • Introduction to Kubernetes Architecture
  • Setting up a Kubernetes Cluster (Minikube or Docker Desktop)

Assignments:

  • Install Minikube or enable Kubernetes in Docker Desktop.
  • Start a local Kubernetes cluster and verify with `kubectl get nodes`.

Week 2: Kubernetes Pods

Objective:

Learn about Kubernetes Pods, the basic unit of deployment in Kubernetes.

Topics Covered:

  • What is a Pod?
  • Creating Pods using `kubectl` commands
  • Understanding Pod Lifecycle and States
  • Pod Management: Scaling Pods and Rolling Updates
  • Working with Multi-container Pods

Assignments:

  • Create a simple Pod using `kubectl run` and verify its status.
  • Scale a Pod and experiment with rolling updates in Kubernetes.

Week 3: Kubernetes Services

Objective:

Learn how to expose applications and services in Kubernetes using Services.

Topics Covered:

  • What is a Kubernetes Service?
  • Types of Services: ClusterIP, NodePort, LoadBalancer, ExternalName
  • Creating and Managing Services
  • Accessing Services within the Cluster and from outside the Cluster

Assignments:

  • Create a Service to expose a Pod using `kubectl expose`.
  • Test the Service by accessing the Pod internally and externally.

Week 4: Kubernetes Deployments

Objective:

Understand how to manage applications in Kubernetes with Deployments.

Topics Covered:

  • What is a Deployment in Kubernetes?
  • Creating and managing Deployments using `kubectl`
  • Rolling Updates and Rollbacks
  • Scaling Deployments
  • Working with Deployment Strategies (Recreate, RollingUpdate)

Assignments:

  • Create a Deployment to deploy a simple web application.
  • Perform a rolling update and rollback on the Deployment.

Week 5: Kubernetes Volumes

Objective:

Learn how to manage persistent storage in Kubernetes using Volumes.

Topics Covered:

  • What is a Kubernetes Volume?
  • Types of Volumes: EmptyDir, HostPath, NFS, PersistentVolume
  • Persistent Volumes (PVs) and Persistent Volume Claims (PVCs)
  • Mounting Volumes in Pods and Deployments

Assignments:

  • Create a Persistent Volume and Persistent Volume Claim in Kubernetes.
  • Mount the volume to a Pod and deploy a stateful application.

Week 6: Kubernetes ConfigMaps and Secrets

Objective:

Learn how to manage configuration and sensitive data in Kubernetes using ConfigMaps and Secrets.

Topics Covered:

  • What is a ConfigMap?
  • Creating and using ConfigMaps in Kubernetes
  • What are Secrets in Kubernetes?
  • Creating and managing Secrets in Kubernetes
  • Injecting ConfigMaps and Secrets into Pods

Assignments:

  • Create a ConfigMap and use it to store configuration data in a Pod.
  • Store sensitive data in Kubernetes Secrets and inject them into Pods.

Week 7: Kubernetes Namespaces and RBAC

Objective:

Understand how to manage and secure Kubernetes resources using Namespaces and Role-Based Access Control (RBAC).

Topics Covered:

  • What is a Namespace in Kubernetes?
  • Managing resources in different namespaces
  • Introduction to RBAC (Role-Based Access Control)
  • Creating and managing Roles and RoleBindings

Assignments:

  • Create a new namespace and deploy applications in it.
  • Set up RBAC policies and assign roles using `kubectl create role` and `kubectl create rolebinding`.

Week 8: Advanced Kubernetes Concepts

Objective:

Learn advanced concepts like Helm, Kubernetes Operators, and Cluster Autoscaling.

Topics Covered:

  • Introduction to Helm: Kubernetes Package Manager
  • Using Helm to deploy applications
  • What are Kubernetes Operators?
  • Implementing Cluster Autoscaling in Kubernetes

Assignments:

  • Install and use Helm to deploy a complex application.
  • Explore Kubernetes Operators and try installing one from the Operator Hub.

Week 9: Kubernetes Monitoring and Logging

Objective:

Learn how to monitor and log Kubernetes clusters for troubleshooting and performance tuning.

Topics Covered:

  • Setting up Prometheus for Kubernetes Monitoring
  • Integrating Grafana with Prometheus
  • Accessing Kubernetes Logs with `kubectl logs`
  • Centralized Logging with Fluentd and Elasticsearch

Assignments:

  • Install Prometheus and Grafana to monitor your Kubernetes cluster.
  • Set up centralized logging using Fluentd and Elasticsearch.

Week 10: Kubernetes Security

Objective:

Understand how to secure a Kubernetes cluster and its resources.

Topics Covered:

  • Securing Kubernetes Cluster with Network Policies
  • Using Kubernetes Pod Security Policies (PSPs)
  • Security Best Practices for Kubernetes Deployments

Assignments:

  • Implement Kubernetes Network Policies to secure Pod communication.
  • Configure Pod Security Policies (PSPs) and enforce security in your cluster.
