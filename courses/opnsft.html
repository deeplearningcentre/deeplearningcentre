OpenShift Learning Plan by Deep Learning Centre Gurgaon

Week 1: Introduction to OpenShift and Containers

Objective:

Understand the basics of OpenShift, containers, and how they fit into modern application deployment.

Topics Covered:

  • What is OpenShift?
  • Containers and Kubernetes basics
  • Why use OpenShift for containerized applications?
  • OpenShift Architecture Overview
  • Setting up OpenShift: Installation options (Minishift, OpenShift CLI)

Assignments:

  • Install Minishift or OpenShift Local and start your first OpenShift cluster.
  • Explore OpenShift Console and the basic CLI commands (`oc` commands).

Week 2: OpenShift Projects and Applications

Objective:

Learn how to create and manage projects in OpenShift and deploy simple applications.

Topics Covered:

  • Understanding OpenShift Projects
  • Creating and managing Projects in OpenShift
  • Deploying simple applications in OpenShift
  • Understanding Deployment Configurations and Builds

Assignments:

  • Create a new OpenShift Project and deploy a simple web application (e.g., a basic Node.js app).
  • Test the application deployment using OpenShift CLI and Console.

Week 3: OpenShift Pods and Deployments

Objective:

Learn about OpenShift Pods and how to manage applications with Deployments.

Topics Covered:

  • Understanding Pods and their lifecycle in OpenShift
  • Creating Pods in OpenShift
  • Scaling Pods and Deployments in OpenShift
  • Managing Rollouts and Rollbacks in OpenShift Deployments

Assignments:

  • Create a Pod and deploy it to an OpenShift Project.
  • Scale a deployment (add replicas) and perform a rolling update of a deployed application.

Week 4: OpenShift Services and Routes

Objective:

Learn how to expose applications in OpenShift using Services and Routes.

Topics Covered:

  • What are Services in OpenShift?
  • Types of Services in OpenShift (ClusterIP, NodePort, LoadBalancer)
  • Creating and Managing Services in OpenShift
  • What are Routes in OpenShift and how to expose services to the outside world

Assignments:

  • Create a Service to expose an application internally.
  • Set up a Route to expose the application externally to the internet.

Week 5: OpenShift Persistent Storage

Objective:

Learn about persistent storage management in OpenShift.

Topics Covered:

  • What is Persistent Storage in OpenShift?
  • Understanding Persistent Volumes (PVs) and Persistent Volume Claims (PVCs)
  • Types of storage supported in OpenShift: NFS, iSCSI, Ceph, etc.
  • Managing and deploying applications with Persistent Storage

Assignments:

  • Create a Persistent Volume (PV) and Persistent Volume Claim (PVC) in OpenShift.
  • Deploy an application that requires persistent storage (e.g., a database).

Week 6: OpenShift Networking and Security

Objective:

Learn how to secure and configure networking in OpenShift.

Topics Covered:

  • Understanding OpenShift Networking concepts
  • Managing Network Policies in OpenShift
  • Securing applications and managing Secrets in OpenShift
  • Service Accounts and Role-Based Access Control (RBAC)

Assignments:

  • Create and apply a Network Policy to restrict access between Pods.
  • Create Secrets and use them in an application to secure sensitive data.

Week 7: OpenShift Scaling and Autoscaling

Objective:

Learn how to scale applications and configure autoscaling in OpenShift.

Topics Covered:

  • What is Horizontal Pod Autoscaling (HPA)?
  • Setting up Autoscaling based on metrics
  • Scaling Applications Manually in OpenShift
  • Vertical Scaling of Pods in OpenShift

Assignments:

  • Enable Horizontal Pod Autoscaling (HPA) on a deployment.
  • Manually scale a deployment and observe the behavior.

Week 8: OpenShift CI/CD Pipelines

Objective:

Learn how to set up Continuous Integration/Continuous Deployment (CI/CD) pipelines in OpenShift using Jenkins.

Topics Covered:

  • Introduction to CI/CD in OpenShift
  • Setting up Jenkins in OpenShift
  • Creating Pipelines with OpenShift Pipelines
  • Integrating GitHub repositories with OpenShift for automated builds

Assignments:

  • Set up Jenkins in your OpenShift Cluster and create a simple pipeline.
  • Integrate a GitHub repository to trigger builds and deployments in OpenShift.

Week 9: OpenShift Monitoring and Logging

Objective:

Learn how to monitor and log applications and resources in OpenShift.

Topics Covered:

  • Setting up Monitoring in OpenShift using Prometheus and Grafana
  • Accessing OpenShift Logs with `oc logs`
  • Using EFK stack (Elasticsearch, Fluentd, Kibana) for centralized logging

Assignments:

  • Set up Prometheus and Grafana in OpenShift for monitoring resources.
  • Install the EFK stack for centralized logging in OpenShift.

Week 10: Advanced OpenShift Topics and Best Practices

Objective:

Explore advanced OpenShift concepts, troubleshooting techniques, and best practices.

Topics Covered:

  • Advanced Networking and Service Mesh in OpenShift
  • OpenShift Operators and Custom Resources (CRDs)
  • OpenShift Troubleshooting techniques and debugging
  • Best practices for securing and scaling OpenShift clusters

Assignments:

  • Configure a Service Mesh with Istio in OpenShift.
  • Deploy an OpenShift Operator and create a Custom Resource (CR).
